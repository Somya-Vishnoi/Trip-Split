                    ⚙️ How TripSplit Works

                    TripSplit uses advanced combinatorial optimization algorithms to build the best itinerary matching your budget constraint.

                    1. Geocoding & Candidate Extraction

                    It geocodes input stops via OpenStreetMap Nominatim and queries OSM Overpass to find stays, restaurants, bars, and attractions within a 2.5km centroid radius.

                    2. Knapsack Dynamic Programming (Stay + Dining)

                    For each stop, TripSplit runs a Constrained 0/1 Knapsack DP solver to choose the stay, restaurants, and attractions that maximize total utility (using ratings, stars, and landmark weights) without exceeding the allocated budget.

                    3. Traveling Salesperson Problem (TSP) Day Routing

                    Sightseeing venues are geographically clustered by day using a Greedy TSP solver, ensuring that the route minimizes travel distance and overhead between sights.

                    4. Parallel Gemini Enrichment

                    Finally, a parallelized Gemini Flash model fetches traveler vibes, local signature dish tips, and coordinates in real-time to enrich the finalized plan stops.
